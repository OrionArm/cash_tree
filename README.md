# Тестовое задание: Full-Stack
---
Условия:

Существует некоторая база данных, представляющая собой агрегатную иерархию однородных элементов
(дерево). В этой базе данных каждый элемент, кроме корня, имеет родительский элемент. Каждый элемент имеет
строковое поле Value. Необходимо добавлять, удалять или изменять элементы в этой базе.

Внимание: существующие связи изменять НЕЛЬЗЯ.

При удалении дочерние элементы также удаляются. Предполагается, что база данных достаточно большая, чтобы
ее нельзя было полностью загрузить для редактирования.

Требования:

Поскольку база данных слишком велика, чтобы загрузить её целиком за один раз, предполагается использовать
некоторый локальный КЭШ. Пользователь может произвольно извлекать элементы из БД в
некоторый локальный кэш, по одному элементу. Если два (или более) элемента в КЭШе связаны
между собой непосредственной связью, они должны автоматически образовывать иерархию, чтобы
пользователь видел их взаимосвязь. Пользователь может
редактировать/удалять/добавлять элементы, работая в КЭШе. (Под редактированием подразумевается
изменение поля Value) Пользователь может сохранить изменения в исходной базе данных
(нажав кнопку «Применить»), применив все внесённые изменения одновременно.

Внимание: кэш может обращаться к базе и элементам базы только в двух случаях: а) когда пользователь
извлекает элемент из базы и б) когда пользователь применяет все изменения. В остальных случаях кэш не
имеет доступа к базе и элементам базы.

Внимание: удалённые элементы не удаляются, а помечаются как удалённые и становятся недоступными для
редактирования (как в базе, так и в кэше).

Внимание: кэш должен стараться не вводить пользователя в заблуждение в условиях ограниченного
доступа к базе данных.

Требования к реализации:

Форма или веб-страница с двумя компонентами TreeView. Один будет отображать/эмулировать БД, другой
будет отображать/эмулировать КЭШ. Соответственно, DBTreeView и CachedTreeView. Оба компонента имеют
контекстное меню или панель кнопок (в зависимости от опыта пользователя), далее просто панель
инструментов. На панели инструментов DBTreeView есть единственная команда — загрузить
выбранный элемент в КЭШ. В этом случае элемент появляется в CachedTreeView. CachedTreeView имеет
следующие команды на панели инструментов: удалить выбранный элемент, задать значение
выбранного элемента, добавить новый дочерний элемент, сохранить изменения в БД. По
умолчанию при запуске должна использоваться стандартная база данных с уровнем вложенности не менее 4. Кнопка reset
возвращает приложение в исходное состояние.
